# 11 轻量测试前后端

## 目标

Navigation 写好后，需要自带一个很小的测试前后端，用于本地开发机验证完整链路。

它参考 BotDog 的导航前后端交互方式，但不能依赖 BotDog 代码、数据库、目录或前端组件。

用途：

- 验证 Navigation 是否能独立建图。
- 验证地图保存和加载。
- 验证重定位 initialpose。
- 验证目标点发布和 `/global_path` 回传。
- 给第三方 Python 后端提供接入样例。

后端使用 Python 标准库 HTTP 服务，不依赖 FastAPI；ROS2 交互由 `rclpy` 完成。

## 后端职责

测试后端只做四类事：

1. 调用 Navigation 兼容脚本。
2. 用 `rclpy` 发布/订阅 ROS2 topic。
3. 通过 HTTP 给测试前端提供操作入口。
4. 通过 WebSocket 推送状态、路径和日志摘要。

不做：

- 用户系统。
- 数据库。
- 任务编排。
- AI/视频。
- 宇树控制。
- 复杂权限。

## HTTP API

建议最小 API：

| Method | Path | 说明 |
|---|---|---|
| `GET` | `/api/status` | 当前系统状态 |
| `POST` | `/api/mapping/start` | 开始建图 |
| `POST` | `/api/mapping/stop` | 停止建图并保存 |
| `GET` | `/api/maps` | 列出地图目录 |
| `POST` | `/api/localization/start` | 加载地图并启动重定位/规划 |
| `POST` | `/api/localization/initialpose` | 发布 `/initialpose` |
| `POST` | `/api/navigation/goal` | 发布目标点 |
| `POST` | `/api/navigation/cancel` | 发布停止 |

示例：

```json
POST /api/navigation/goal
{
  "x": 1.2,
  "y": 3.4,
  "z": -0.83,
  "yaw": 0.0,
  "frame_id": "map"
}
```

## WebSocket

建议一个 WebSocket：

```text
/ws
```

事件类型：

```json
{
  "type": "nav.status",
  "data": {
    "status": "moving",
    "message": "导航中"
  }
}
```

推荐事件：

- `system.status`
- `mapping.status`
- `localization.status`
- `nav.status`
- `nav.path`
- `robot.pose`
- `logs.tail`

第一版实现使用 `GET /api/events` 的 Server-Sent Events 推送状态，前端用 `EventSource` 接收。后续如果测试面板需要更复杂的双向交互，再替换为 WebSocket。

## ROS2 topic

测试后端应使用 Navigation 公共 topic：

发布：

```text
/clicked_point
goal_yaw
/nav_start
/nav_stop
/initialpose
```

订阅：

```text
/global_path
/nav_status
/lio/cloud_world
/lio/odom
TF map -> base_footprint
```

这些 topic 名称后续可以通过配置或 remap 改成 `/nav/...` 命名空间。

## 前端页面

测试前端只需要一个页面：

```text
顶部状态栏:
  ROS 连接状态、当前模式、地图名、定位状态

建图区域:
  地图名输入
  开始建图
  停止建图
  地图文件列表

重定位区域:
  选择地图
  启动重定位
  输入 x/y/z/yaw
  发布 initialpose

导航区域:
  输入目标 x/y/z/yaw
  发送目标
  停止

显示区域:
  当前路径点数
  点云俯视预览
  当前机器人位姿
  日志尾部
```

第一版已实现 canvas 2D 点云预览。点云来源：

```text
/lio/cloud_world
/terrain_map
/nav/local_obstacles
```

测试面板只做抽样预览，默认每帧最多显示 1500 个点；精细点云检查仍应使用 RViz。

## 与 BotDog 的区别

测试前后端是 Navigation 自测工具，不替代 BotDog。

| 项 | BotDog | Navigation 测试前后端 |
|---|---|---|
| 业务功能 | 完整业务系统 | 只测导航 |
| 数据库 | 有 | 无 |
| 用户权限 | 有 | 无 |
| 视频/AI | 有 | 无 |
| 宇树控制 | 有 | 无 |
| 目标 | 产品前后端 | 开发验证和接入样例 |

## 验收标准

轻量测试前后端完成后，应能独立完成：

1. 打开页面看到 ROS/Navigation 状态。
2. 输入地图名并开始建图。
3. 停止建图后看到 `map.pcd`、`ground.pcd`、`footprint_fill.pcd`。
4. 选择地图启动重定位/规划。
5. 发布 initialpose。
6. 发布目标点。
7. 收到 `/global_path` 并显示路径点数。
8. 收到 `/nav_status` 并显示状态。

## 当前验证结果

已完成：

- `GET /api/status` 返回 ROS2 桥接状态。
- `GET /api/maps` 返回地图目录列表。
- 前端首页可访问。
- `POST /api/localization/initialpose` 可发布 `/initialpose`。
- `POST /api/navigation/goal` 可发布 `/clicked_point`、`goal_yaw` 和 `/nav_start`。
- `POST /api/navigation/cancel` 可发布 `/nav_stop` 和 `/nav_start=false`。
- 临时发布 `/lio/cloud_world` 后，`GET /api/status` 可返回点云抽样数据。

未做真机验证：

- 未启动真实建图，避免在未连接雷达和算法包未完全联调时误触发硬件链路。
- 未验证 `/global_path` 回传，因为当前没有运行规划链路。
