# 08 健康检查与验收清单

## 建图健康检查

启动后必须满足：

- `/livox/lidar` 有数据。
- `/livox/imu` 有数据。
- Super-LIO 日志出现地图初始化完成。
- `/lio/cloud_world` 有数据。
- terrain_analysis 进程存活。
- `<map_dir>/.ground_generation_started` 已写入。

停止后必须满足：

- `map.pcd` 存在且非空。
- `ground.pcd` 或 `*_ground.pcd` 存在且非空。
- `footprint_fill.pcd` 或 `*_footprint_fill.pcd` 存在时非空。
- `metadata.yaml` 存在。
- 旧进程无残留。

## 重定位健康检查

- Livox driver 正常。
- relocation node 正常。
- `/initialpose` 有订阅者。
- BotDog 后端 initialpose publisher 可匹配订阅者。
- 发布 `/initialpose` 后 `/lio/odom` 有输出。
- TF `map -> base_footprint` 可查询。

## 规划健康检查

- `map.pcd`、`ground.pcd` 加载成功。
- 如有 `footprint_fill.pcd`，planground 加载成功。
- `/weighted_ground` 发布过。
- `/clicked_point` 发布后 `/global_path` 输出。
- 目标点 z 靠近 ground 点云。
- 机器人当前位置在 ground 点云附近。

## BotDog 联动健康检查

- 前端建图按钮能启动建图。
- 前端停止建图后能看到地图文件。
- 前端重定位流程能进入 initialpose 标记模式。
- 发布 initialpose 后定位状态恢复。
- 前端点击巡检点 go-to 后，后端发布 `/clicked_point` 和 `goal_yaw`。
- 后端能收到 `/global_path` 并通过 WebSocket 转发。
- `/nav_status` JSON 能更新前端状态。

## 常用检查命令

```bash
ros2 topic list
ros2 topic echo /livox/imu --once --qos-reliability best_effort
ros2 topic echo /livox/lidar --once --qos-reliability best_effort
ros2 topic echo /lio/odom --once
ros2 topic echo /global_path --once
ros2 topic info /initialpose -v
ros2 run tf2_ros tf2_echo map base_footprint
```

## 不通过时优先排查

1. Livox 本机 host IP 是否和雷达同网段。
2. `ROS_DOMAIN_ID` 是否一致。
3. 是否 source 了正确 workspace。
4. 是否有旧进程残留占用 topic。
5. 地图目录是否传错。
6. `map.pcd` 是否用于 relocation，`ground.pcd` 是否用于 planner。
7. goal z 是否偏离 ground 点云。
