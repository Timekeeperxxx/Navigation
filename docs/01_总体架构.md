# 01 总体架构

## 目标

把现有导航能力封装成一套低耦合、可被不同上层系统稳定调用的 ROS2 导航系统。`BotDog-jetson-main` 只作为当前已有前后端的参考接入方，不作为 Navigation 的内部依赖。

- `Superlio-main`：Livox 驱动、Super-LIO 建图/重定位、terrain_analysis 地形分析。
- `dddmr_global_planner-main`：静态地图/地面点云全局规划。
- `BotDog-jetson-main`：参考前后端业务、地图管理、任务管理、WebSocket 转发、宇树 SDK 控制链路。

本次不是重写算法，而是整理联动关系、入口、配置、文件约定和健康检查。

## 推荐分层

```text
上层应用层
  可以是 BotDog、其他 Python 后端、Web 后端、调度系统或测试脚本
  只能通过 Navigation 暴露的 ROS2/CLI/API 边界接入

Navigation API/Bridge 层
  提供稳定 ROS2 topic/service/action
  可选提供 CLI wrapper 或 HTTP/WebSocket 网关
  不依赖具体上层项目目录

Navigation bringup 层
  新增本地封装包和脚本
  统一启动 Livox、Super-LIO、terrain_analysis、global_planner
  统一地图路径、日志、PID、ready 文件

算法/驱动层
  livox_ros_driver2
  super_lio
  terrain_analysis
  global_planner

硬件控制层
  BotDog 现有宇树 SDK / cmd_vel 桥
```

## 数据流

建图阶段：

```text
Livox MID360
  -> /livox/lidar, /livox/imu
  -> Super-LIO mapping
  -> /lio/odom, /lio/cloud_world, map.pcd
  -> terrain_analysis
  -> ground.pcd, footprint_fill.pcd
```

运行阶段：

```text
map.pcd
  -> Super-LIO relocation
  -> TF map -> base_footprint, /lio/odom

ground.pcd + footprint_fill.pcd
  -> global_planner
  -> /global_path

实时点云 / 局部障碍
  -> terrain_analysis / local_map / perception layer
  -> 动态可通行区域或障碍层
  -> 局部避障或触发重规划

BotDog 后端
  -> /clicked_point + goal_yaw
  -> global_planner
  -> /global_path
  -> BotDog 前端显示 / 宇树控制链路
```

## 低耦合最小改动策略

Navigation 应优先提供通用契约。为了兼容当前 BotDog，可保留以下 ROS2 topic 名称，但这些 topic 属于 Navigation 的对外接口，不属于 BotDog 私有接口：

- 单点导航仍由后端发布 `/clicked_point` 和 `goal_yaw`。
- 任务导航仍由后端生成 `data/nav_runtime/current_task.json` 并发布 `/nav_start true`。
- 重定位仍由后端发布 `/initialpose`。
- 后端仍订阅 `/global_path`、`/nav_status`、`/lio/cloud_world`。

BotDog 适配只能出现在 adapter/wrapper 层，不能进入核心 bringup：

```text
Navigation core:
  nav_bringup launch/config
  robot_nav_interfaces
  map/runtime/log 管理

Optional adapter:
  botdog-compatible shell wrapper
  botdog-compatible topic names
  botdog-compatible ready/log 文件
```

## 不建议先做的事

- 不先 Docker 化。
- 不先重写 Super-LIO ESKF、OctVoxMap、relocation 核心逻辑。
- 不先重写 global_planner 的 A*/Hybrid A*。
- 不把 BotDog 前端直接改成调用 ROS2。
- 不让前端/后端直接依赖算法包内部文件结构。
- 不把动态避障硬塞进静态全局规划器；应拆成局部避障、动态障碍层、重规划触发三部分逐步实现。
