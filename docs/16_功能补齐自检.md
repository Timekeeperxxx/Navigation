# 功能补齐自检

本文记录当前 Navigation 工作区内各功能的实现状态、证据和缺口。判断依据以当前源码、安装规则和可运行检查为准。

## 总结

| 功能 | 状态 | 当前证据 | 主要缺口 |
| --- | --- | --- | --- |
| 建图 | 已接入，可测试 | `nav_bringup/launch/mapping.launch.py` 直接启动 `livox_ros_driver2`、`nav_lio/super_lio_node`、`nav_terrain_analysis`、`nav_save_terrain_map` | 需要继续用真实雷达验证长时间建图稳定性 |
| terrain 分层保存 | 已接入，可测试 | `nav_terrain` 内置 terrain 分析和保存节点，保存目录兜底已改为 `/tmp/navigation_terrain_map` | ground 对桌底、机箱上表面的误判仍是待查阅问题 |
| 重定位 | 已接入，可测试 | `localization.launch.py` 启动 `nav_lio/relocation_node`，加载 `map.pcd`，并发布 RViz ground map | `base_link -> base_footprint` 的高度/姿态语义仍需结合机器人实物确认 |
| 全局规划 | 已接入，可测试 | `planning.launch.py` 启动 `global_planner_node`，订阅 `/clicked_point`、`/goal_pose`，发布 `/global_path` | 起点 frame 当前使用 `base_footprint`，是否改为 `base_link` 需要后续决策 |
| DWA-aware 全局路径 | 已接入，可测试 | `global_planner_node` 内创建 `/get_dwa_plan`，`clicked_dwa_planner_client.py` 可由 RViz 点击触发，发布 `/awared_global_path` | 目前仍基于静态/规划图，不等同于完整 MPC/DWA 局部规划 |
| 动态避障 | 已接入第一阶段，可离线测试 | `dynamic_avoidance_monitor.py` 订阅 `/global_path` 和 `/nav/local_obstacles`，发布 `/prune_plan`、`/nav/local_path`、`/nav/obstacle_status`、`/nav/replan_request`、`/cmd_vel_safe` | 真机雷达动态障碍提取链路还需接入；当前无雷达时可用 `local_obstacle_simulator.py` 仿真 |
| 局部规划 | 已接入轻量版本，默认关闭 | `nav_path_follower.py` 根据 `/nav/local_path` 或 `/global_path` 生成保守 `/cmd_vel`，并监听动态避障状态停车/限速 | 这不是源码完整轨迹采样 + critics 局部规划；真机控制前仍需调参和安全验收 |
| 巡检任务执行 | 无底盘版本已接入 | `waypoint_navigator_from_json.py` 发布航点到 `/clicked_point`，`waypoint_progress_monitor.py` 根据 TF 发布 `/waypoint_reached` | 真实到点依赖机器人运动；无底盘环境只能验证任务状态机和路径发布 |
| 底盘控制 | 暂不实现 | 用户确认当前没有底盘控制环境 | 后续需要底盘接口、控制器、急停和速度限制策略 |
| RViz 调试 | 已接入 | `navigation_debug.rviz` 显示 map、ground、planground、`/global_path`、`/awared_global_path`、TF | 视觉布局可继续按测试反馈微调 |
| 前端/runtime | 部分完成 | `nav_runtime` 调用 Navigation 自己的 `nav_bringup` launch；测试前端可触发建图/导航 | 前端任务编排和运行状态显示还不是最终产品形态 |

## 当前可测命令

普通重定位 + 全局规划：

```bash
ros2 launch nav_bringup navigation.launch.py \
  nav_lio_map_dir:=/path/to/scene \
  map_name:=map.pcd \
  map_pcd:=/path/to/scene/map.pcd \
  ground_pcd:=/path/to/scene/terrain_map_xxx_ground.pcd \
  planground_pcd:=/path/to/scene/terrain_map_xxx_base_footprint_fill.pcd \
  rviz:=true
```

开启 DWA-aware 路径和无底盘巡检状态机：

```bash
ros2 launch nav_bringup navigation.launch.py \
  nav_lio_map_dir:=/path/to/scene \
  map_name:=map.pcd \
  map_pcd:=/path/to/scene/map.pcd \
  ground_pcd:=/path/to/scene/terrain_map_xxx_ground.pcd \
  planground_pcd:=/path/to/scene/terrain_map_xxx_base_footprint_fill.pcd \
  enable_dwa_client:=true \
  enable_waypoint_navigator:=true \
  enable_waypoint_monitor:=true \
  waypoints_file:=/path/to/waypoints.json \
  rviz:=true
```

开启动态避障离线仿真和轻量路径跟随：

```bash
ros2 launch nav_bringup navigation.launch.py \
  nav_lio_map_dir:=/path/to/scene \
  map_name:=map.pcd \
  map_pcd:=/path/to/scene/map.pcd \
  ground_pcd:=/path/to/scene/terrain_map_xxx_ground.pcd \
  planground_pcd:=/path/to/scene/terrain_map_xxx_base_footprint_fill.pcd \
  launch_livox:=false \
  launch_relocation:=false \
  enable_dynamic_avoidance:=true \
  enable_obstacle_simulator:=true \
  enable_path_follower:=true \
  enable_static_base_tf:=true \
  rviz:=true
```

无雷达时可手动放置一个仿真障碍：

```bash
ros2 topic pub --once /nav/sim_obstacle_point geometry_msgs/msg/PointStamped \
  "{header: {frame_id: map}, point: {x: 1.0, y: 0.0, z: 0.0}}"
```

清空仿真障碍：

```bash
ros2 topic pub --once /nav/clear_sim_obstacles std_msgs/msg/Bool "{data: true}"
```

如果只是离线验证 topic 逻辑且没有 TF，可临时给 `dynamic_avoidance_monitor.py` 设置 `allow_path_start_without_tf:=true`。真实运行不要打开这个兜底。

启动巡检任务：

```bash
ros2 topic pub --once /nav_start std_msgs/msg/Bool "{data: true}"
```

查看巡检进度：

```bash
ros2 topic echo /nav/waypoint_progress
```

## 后续必须补齐的部分

1. 真机动态障碍输入：把实时 LiDAR/terrain/local map 的障碍提取结果接到 `/nav/local_obstacles`。
2. 完整局部规划器：如需恢复源码级轨迹采样 + critics，需要继续迁移 `dddmr_local_planner`、`mpc_critics` 和 `trajectory_generators`。
3. 底盘控制接口：等底盘环境可用后，再确认 `/cmd_vel`、`/cmd_vel_safe` 或机器人实际控制接口的最终接线。
4. 到点判定策略：现在按 TF 和半径判断，后续要结合底盘状态、速度、姿态和任务动作结果。
5. 真实场景验收：建图、重定位、规划、动态避障、巡检都需要在同一张新建地图上连续跑通。
