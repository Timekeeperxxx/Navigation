# 04 重定位与规划流程

## 目标

加载已保存地图，完成 Super-LIO 重定位，并启动 global_planner 使用 `ground.pcd` 和 `footprint_fill.pcd` 规划。

## 输入

场景目录：

```text
Scene001_xxx/
  map.pcd
  ground.pcd
  footprint_fill.pcd
  metadata.yaml
```

BotDog 当前会从场景目录中查找：

- `map.pcd` 或 `*map.pcd`
- `ground.pcd` 或 `*ground.pcd`
- `footprint_fill.pcd` 或 `*footprint_fill.pcd`

这个兼容逻辑应保留。

## 启动顺序

1. 清理旧的 Livox、relocation、global_planner、p2p_move_base、cmd_vel 进程。
2. source ROS2 和 Navigation workspace。
3. 启动 Livox MID360 驱动。
4. 启动 Super-LIO relocation，加载 `map.pcd`。
5. 等待 relocation 进入 `/initialpose` 等待状态。
6. 启动必要 TF。
7. 启动 global_planner，传入：
   - `map_dir:=map.pcd`
   - `ground_dir:=ground.pcd`
   - `planground_dir:=footprint_fill.pcd`
8. 等待 global_planner 静态图加载完成。
9. 写入 `navigation_ready.json`。
10. BotDog 前端发布重定位点，后端发布 `/initialpose`。
11. relocation 成功后输出 TF 和 `/lio/odom`。
12. BotDog 发送 `/clicked_point`，global_planner 发布 `/global_path`。

## ready 判断

当前脚本通过日志判断：

- `Map pointcloud size after down size`
- `Ground pointcloud size after down size`
- `Planground pointcloud size after down size`
- `Publish weighted ground point cloud.`

第一阶段可以继续保留日志判断，同时增加 topic/service 级 ready：

- `/weighted_ground` 发布过。
- `/global_path` publisher 存在。
- TF `map -> base_footprint` 可查询。
- `/lio/odom` 在 initialpose 后可输出。

## 目标点约束

BotDog 当前发布真实 ground z，避免 global_planner 3D KD-tree 半径搜索失败。

继续保留：

```text
ROS_NAV_GOAL_Z_SEARCH_OFFSET_M=0.0
```

不要随意抬高目标 z。导航点应通过 ground snapping 落到 `ground.pcd` 附近。
