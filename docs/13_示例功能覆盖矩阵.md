# 13 示例功能覆盖矩阵

## 目标

本文逐项对照 `Superlio-main` 和 `dddmr_global_planner-main` 中已经存在的能力，确认 Navigation 重写封装时是否覆盖、如何覆盖，以及属于 core、adapter、工具还是可选扩展。

原则：

- 不遗漏示例工程已有功能。
- 不把所有功能都硬塞进第一阶段主链路。
- 核心链路必须完整健康。
- 可选能力要预留接口和配置，不静默丢弃。

## 覆盖状态定义

| 状态 | 含义 |
|---|---|
| 核心必做 | Navigation 第一阶段主链路必须支持 |
| adapter 兼容 | 不进 core，但提供脚本、remap 或兼容输出 |
| 工具保留 | 作为调试/编辑/转换工具保留入口 |
| 可选扩展 | 后续可接入，第一阶段不强制启动 |
| 仅记录 | 了解存在，但当前目标不纳入实现 |

## 当前文档补漏结论

此前文档已覆盖：

- 主建图链路。
- 主重定位与规划链路。
- 地图目录格式。
- BotDog 低耦合接入。
- 动态避障第一阶段。
- 轻量测试前后端。

本次补充后，新增覆盖：

- Livox 多设备/多格式驱动能力。
- Super-LIO 扩展输出和多数据集配置。
- terrain_analysis 保存服务、清图、动态障碍参数。
- FASTLIO2 的 PGO、localizer、HBA 可选扩展。
- global_planner action、DWA-aware、ROI、polygon、waypoint、PCD 编辑、occupancy 转换工具。
- 外部依赖边界。
