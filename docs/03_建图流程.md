# 03 建图流程

## 目标

本地开发阶段先保留 BotDog 当前建图入口：

```text
前端 -> BotDog 后端 -> scripts/start_mapping.sh <map_dir>
```

内部逐步替换为统一 Navigation bringup，但对 BotDog 外部行为保持一致。

## 输入

| 输入 | 来源 |
|---|---|
| `map_dir` | BotDog 后端传入的场景目录 |
| Livox MID360 点云 | `/livox/lidar` |
| Livox MID360 IMU | `/livox/imu` |
| 雷达 IP | `192.168.123.179` |
| 本机 host IP | 配置项，不写死 |

注意：`192.168.123.179` 是雷达 IP，不是本机 IP。host IP 必须按本机网卡配置。

## 输出

每次建图结束后，场景目录必须至少包含：

```text
map.pcd
ground.pcd 或 *_ground.pcd
footprint_fill.pcd 或 *_footprint_fill.pcd
metadata.yaml
```

语义：

- `map.pcd`：Super-LIO 重定位用。
- `ground.pcd`：global_planner 地面静态层用。
- `footprint_fill.pcd`：global_planner 的 planground/辅助可通行点用。
- `metadata.yaml`：记录地图、雷达、frame、生成时间、版本信息。

## 建图启动顺序

推荐顺序保持当前实践：

1. 停止正在运行的导航/重定位/规划进程。
2. 暂停 BotDog 后端 ROS bridge，降低 DDS 干扰。
3. 启动 Livox MID360 驱动。
4. 等待 `/livox/lidar`、`/livox/imu` 可用。
5. 启动 Super-LIO 建图。
6. 等待 Super-LIO 地图初始化完成。
7. 启动 terrain_analysis 与保存节点。
8. 写入 ready 标记，通知 BotDog 建图链路已启动。

## 建图停止顺序

停止顺序必须谨慎，避免 PCD 未落盘：

1. 先停止 terrain_analysis，确保 ground/footprint 文件保存。
2. 再停止 Super-LIO node，确保 `map.pcd` 保存。
3. 再停止 Super-LIO launch 剩余进程。
4. 最后停止 Livox 驱动。
5. 检查 PCD 文件。
6. 写入 metadata。
7. 恢复 BotDog ROS bridge。

## 保留当前 ready 机制

BotDog 当前依赖：

```text
<map_dir>/.ground_generation_started
```

第一阶段应继续写这个文件，避免改动后端判断逻辑。

后续可以补充：

```text
data/nav_runtime/mapping_ready.json
```

但不替代现有 ready flag。
